# Gallery Asset Standards (rules_gallery_standards.md)

**Authority:** Enforced on all static gallery assets in `assets/gallery_data/`.

## 1. The Scientific Triplet (Mandatory Files)

Every gallery bundle directory MUST contain exactly:

- **`recipe_manifest.yaml`**: The executable logic (info, plots, wrangling).
- **`example_data.tsv`**: A representative dataset (tab-separated).
- **`recipe_meta.md`**: High-density scientific guidance and author metadata.
- **`preview_plot.png`**: The rendered evidence of the visualization.

## 2. Taxonomy Requirements (6-Axis, ADR-063)

The `info:` block in `recipe_manifest.yaml` MUST define all 6 taxonomy fields:

```yaml
info:
  name: Recipe Name
  submission_date: 'YYYY-MM-DD'
  family: Distribution         # 📊  see §2a
  pattern: 1 Numeric, 1 Categorical  # 🔢  see §2b
  difficulty: Intermediate     # 📈  Simple | Intermediate | Advanced
  geom: geom_violin            # ⚙️  exact plotnine function name, lowercase
  show: distribution           # 🎯  see §2c
  sample_size: medium+         # 📏  see §2d
```

**Canonical reference:** `assets/gallery_data/TAXONOMY_CHEATSHEET.md` — single source of truth for all allowed values and icon assignments. Do NOT invent new values; add them to the cheatsheet first if needed.

### §2a — 📊 Family (Purpose)
`Comparison` · `Correlation` · `Distribution` · `Evolution` · `Part-to-Whole` · `Ranking`

### §2b — 🔢 Data Pattern
`1 Numeric` · `1 Numeric, 1 Categorical` · `1 Numeric, 2 Categorical` · `1 Numeric, 2 Categorical (Faceted)` · `2 Numeric` · `2 Numeric, 1 Categorical` · `2 Numeric, 1 Categorical (Faceted)` · `3 Numeric (Matrix)` · `1 Year, 1 Numeric` · `1 Year, 1 Numeric, 1 Categorical`

### §2c — 🎯 What it Shows
`change` · `distribution` · `frequency` · `normality` · `proportion` · `ranking` · `relationship` · `trend` · `uncertainty`

### §2d — 📏 Sample Size
`any` · `individual points` · `medium+` · `large`

### §2e — ⚙️ Primary Geom
The exact plotnine function name (lowercase, underscore). Must be the first `geom_*` or `stat_*` layer in the recipe. See `TAXONOMY_CHEATSHEET.md` for the full allowed list.

## 3. recipe_meta.md Standard Format (ADR-061, ADR-063)

All `recipe_meta.md` files follow a 3-tier structure:

```markdown
## [Recipe Name]

> 📊 Family · 🔢 Data Pattern · 📈 Difficulty · ⚙️ geom_name · 🎯 show_value · 📏 sample_size_value

### Section Heading
…
```

- `##` = recipe name (h2) — NOT `# Recipe Metadata: …`
- `>` blockquote = 6-field taxonomy tag strip on the line immediately after the h2
- `###` = content section headings (Suitability, Data Schema, Transformation Logic, etc.)
- Icon assignments MUST match the cheatsheet exactly: `📊 🔢 📈 ⚙️ 🎯 📏`

## 4. VizFactory Layer Format

Recipes use the VizFactory manifest format. **Do NOT use `geom:` as a layer key** — use `name:`. Mapping belongs at the global `mapping:` key under the plot, not per-layer.

Correct:
```yaml
plots:
  my_plot:
    mapping:
      x: column_a
      y: column_b
    layers:
      - name: geom_point
```

Wrong (both errors):
```yaml
plots:
  my_plot:
    layers:
      - geom: geom_point        # ← wrong key
        mapping: {x: column_a}  # ← mapping belongs at plot level
```

## 5. The Pivot-Index Integrity Gate

- `gallery_index.json` is the sole source of truth for the UI.
- Direct editing of the index is FORBIDDEN.
- Regenerate via `assets/scripts/refresh_gallery.py` (or `gallery_manager.rebuild_index()`).
- Any bundle missing a required taxonomy field will be excluded from the index.

## 6. Preview PNG Generation

- `preview_plot.png` is generated by `libs/viz_gallery/assets/generate_previews.py`.
- Run after adding or updating any recipe:
  ```bash
  .venv/bin/python libs/viz_gallery/assets/generate_previews.py [--only name] [--force]
  ```
- The script reads `recipe_manifest.yaml` + `example_data.tsv`, renders the first `plot_id` via VizFactory, and writes `preview_plot.png`.
- Do NOT commit a bundle without a valid `preview_plot.png`.
