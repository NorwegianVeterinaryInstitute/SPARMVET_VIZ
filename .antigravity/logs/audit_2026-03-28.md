### 1. Test Runner Migration (ADR-005, ADR-016)
- **Migration**: Refactored the legacy `test_wrangler.py` into the **Universal Wrangler Runner** (`wrangler_debug.py`).
- **Rationale**: The new runner supports strictly declarative YAML manifests with argparse overrides, enabling rapid out-of-core testing of pipeline segments.

### 2. The 1:1:1 Naming Law (Section 8)
- **Codification**: Established a mandatory 1:1:1 mapping between Implementation, Configuration, and Evidence.
- **Structure**:
    - **Logic**: `@register_action("action_name")`
    - **Manifest**: `libs/transformer/tests/data/action_name_manifest.yaml`
    - **Data**: `libs/transformer/tests/data/action_name_test.tsv`
- **Automation**: Created `test_decorator_suite.py` to enforce this mapping via the live action registry.

### 3. Categorical vs. String Standard (Section 12)
- **Standard**: "Clean-then-Cast" workflow.
- **Logic**: All wrangling actions (cleaning/stripping/regex) must operate on raw `pl.String`. Casting to `pl.Categorical` is reserved as a **Terminal Operation** in the `output_fields` contract.
- **Performance**: Optimized for string manipulation speed while maintaining relational join integrity via encoded categories.
