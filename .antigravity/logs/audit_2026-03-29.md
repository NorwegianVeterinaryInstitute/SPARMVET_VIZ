## тЪая╕П Operational Note: "Waiting for Poll" Resolution
**Context:** Multiple background shell commands (`command_status`) returned `Status: RUNNING, No output`
even after the Python scripts had completed and written all expected files to disk.

**Root cause:** Platform-level terminal buffer/polling latency in the Antigravity IDE.
The process finished but the stdout handle was not flushed to the agent's polling channel
within the check window. This caused several false "still running" states (up to 8+ minutes).

**Resolution applied:** Artist Law evidence-driven verification.
Instead of waiting for terminal status, the agent used `find_by_name` and `view_file`
to confirm artifact presence on disk (TSVs, PNGs). This is the correct pattern to follow
when terminal polling hangs.

**Training Context:** When `command_status` returns `Status: RUNNING, No output` for >3 minutes
on a task that writes output files, switch to disk-state verification immediately.
Do NOT continue polling indefinitely тАФ check artifacts instead.
