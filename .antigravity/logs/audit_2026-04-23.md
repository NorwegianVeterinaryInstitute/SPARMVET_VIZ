## Decisions Made

### ADR-043: Unified Home Theater

**Trigger:** The "Analysis Theater / Viz" nav item was identified as dead weight — it shared the same `dynamic_tabs()` render function as "Home" and differed only in the header label. The `analysis_groups` manifest tabs were appended identically to both modes.

**Decision:**
- Eliminate "Analysis Theater / Viz" nav item entirely.
- Home becomes the sole results mode. Its tab structure derives **exclusively** from manifest `analysis_groups` (no hardcoded tabs, Inspector tab removed — it has no manifest backing and violates ADR-003/ADR-004).
- Introduce a **Tier Toggle** radio-button strip (T1 Raw / T2 Reference / T3 Wrangling / T3 Plot) replacing `ref_tier_switch` + `view_toggle`. T3 states are persona-gated (≥ `pipeline_exploration_advanced`).
- **Comparison Mode** is a separate, persona-gated toggle (≥ advanced). When ON: 2-column split (T2 reference left, T3 active right). Option A chosen (independent of Tier Toggle).
- Left sidebar filter widgets are **context-reactive to the active plot sub-tab** — regenerated on sub-tab change, scoped to that plot's `plot_spec` aesthetics.
- Plot sub-tabs wrapped in a **collapsible accordion** (default expanded). Data preview in a **separate collapsible accordion below**. Collapse state persists across sub-tab navigation.
- Column picker enforced at `width: 100%`, no multi-row wrapping.
- For lower personas (no T3 access): Tier Toggle shows T1/T2 only; T3 buttons hidden. T3 silently pre-fills from T2 (functionally identical, never displayed).

### ADR-044: Persona-Gated Audit Stack & Right Sidebar Visibility

**Trigger:** For `pipeline_static` and `pipeline_exploration_simple` personas, the right sidebar (Audit Stack) contains no actionable information — T3 ≡ T2 silently, there is no sandbox to inspect. Displaying it wastes screen real estate and misrepresents the interaction model.

**Decision:**
- Right sidebar **hidden entirely** for `pipeline_static` and `pipeline_exploration_simple`. The layout element is excluded (not CSS-hidden) so the theater column expands to fill full width.
- Right sidebar **visible** for ≥ `pipeline_exploration_advanced`: shows T2 Violet blueprint nodes + T3 Yellow sandbox nodes + `btn_apply` + `btn_revert`.
- T3 recipe silently pre-fills from T2 for all personas (protecting plot formatting). Lower personas never see this machinery.
- T2 Violet nodes are visible only when the right sidebar is visible (i.e., only for advanced+ personas who can meaningfully interact with them).

## Session Block 2 — Server Decomposition Architecture

### ADR-045: Server Decomposition — Handlers Directory & Manifest Navigator Module

**Trigger:** `app/src/server.py` reached 2,362 lines across five functionally distinct concerns with no structural separation. Phase 21 is about to add more Home Theater logic. Decomposition now prevents further monolith growth.

**Key decision: Option B — `app/modules/manifest_navigator.py`** (not co-located with handlers)

The five pure manifest introspection helpers (`_build_sibling_map`, `_build_schema_registry`, `_build_lineage_chain`, `_load_fields_file`, `_resolve_fields_for_schema`) are extracted to `app/modules/manifest_navigator.py` with public names (drop `_` prefix). Rationale:
- They are pure functions with zero Shiny dependency.
- Needed by the Field Gap Analysis tool (Phase 20), headless debug scripts, and DevStudio.
- Placing them in a handler file would create import hazards (handler files carry `@render.*` side-effects when imported).
- Consistent with the existing `app/modules/` pattern for importable, testable components.

**The Two-Category Law** (architectural lock):
- Pure introspection → `app/modules/` (importable anywhere)
- Shiny wiring → `app/handlers/` (never imported by non-Shiny contexts)

**Five handler modules** created in `app/handlers/`: `home_theater.py`, `audit_stack.py`, `blueprint_handlers.py`, `gallery_handlers.py`, `ingestion_handlers.py`. Each exposes `define_server(input, output, session, *, ...)` with keyword-only dependencies.

**`server.py` target:** ≤ 150 lines — imports, shared state/calcs, shared utils, five delegation calls only.

## Technical Meta-Analysis: Session Friction Points

### 1. Inexplicit Manifest Rules
- **"Where was the gene column filtered?"**: Highlighted that the `final_contract` as a **Strict Projection Guard** was not explicitly documented.
- **"Why is Country not materializing?"**: Showed a conflict between the engine's implicit dropping of columns and the user's expectation of a "pass-through" behavior.
- **"How is sequence_type included while not in raw data?"**: Demonstrated that **Resilient/Silent Ingestion** (ADR-013) can mask intent, leading to confusion.

### 2. Implicit Knowledge Applied (Not in Protocol)
- **ADR-041 Compliance**: Executed the "Mirrored Directory" standard (nested fragments in `basename/` folders) which is currently only in structural rules, not testing guides.
- **YAML Quoting Nuances**: Applied manual quoting to the `on` keyword in joins (reserved boolean in YAML).
- **Library Script Discovery**: Manually mapped the `@verify` protocol to `debug_wrangler.py` and `debug_assembler.py` in the `libs/transformer/tests` directory.

### 3. Missing Structural Instructions
- **Scientific Audit Mandate**: Lack of an explicit instruction to automatically materialize Tier 1 (Atomic Wrangling) for download/audit of raw cleaning.
- **Biological Typing Standard**: Need for a rule that discrete variables (Year, ST) must be cast to `Int` or `String` in Tier 2 to satisfy Plotnine aesthetic requirements.
- **Whitelisting Policy**: Clarification needed that manifests are *exclusive* (only listed columns survive).
