### Notable decisions and deviations from the original plan

- **24-B kwarg surface narrowed.** The plan in `tasks_phase24.md` listed seven reactive kwargs. Reading the actual code showed the session block reads only three: `session_manager`, `current_persona`, `home_state`. Final signature reflects that.
- **24-C inline `_op_label` duplicate (resolved in 24-D).** During 24-C the helper was inlined in `export_handlers.py` to keep the move purely mechanical (no new shared-module dependency). 24-D folded a prep step that lifted `_op_label` to `app/modules/t3_recipe_engine.py` so both handlers now import from one place.
- **24-D filter UI + T3 audit kept in one file.** ADR-051 originally anticipated two files (`t3_audit_handlers.py` plus a filter UI handler). I kept them together as `filter_and_audit_handlers.py` because `_filter_apply`, `_col_drop_to_audit`, and `_handle_propagation_confirm` all share `_propagation_scratch` and call `_open_propagation_modal`. Splitting would force exposing scratch state across module boundaries — a worse outcome than the larger file. Documented in ADR-051 and tasks_phase24.md.
- **24-D kwarg surface large but unavoidable.** 14 kwargs to `define_filter_audit_server` (4 reactive.Values, 6 closure callables from `define_server`, plus `input/output/session/tier_toggle/active_home_subtab/safe_input`). The closures could not be lifted without restructuring `define_server`'s internal ordering. Accepted the kwarg breadth as the cost of an otherwise mechanical move.

### Bonus fix in this run

`166ff82 fix(tests): module-scoped page fixture + idempotent _load_project` — committed early in the run (before 24-A landed on 2026-04-30, included here for completeness). Diagnosed the smoke suite as flake-prone (~33% failure rate) due to Shiny session accumulation on resource-constrained machines: every test was creating a fresh browser context → fresh Shiny WebSocket session → repeated T1 materialisation timeouts. Fixed with two changes: (1) module-scoped `page` and `browser` fixtures in `app/tests/conftest.py`, (2) idempotent `_load_project` in `test_shiny_smoke.py` that skips navigation when Quality Control nav is already visible. Suite went from 35-110s flaky to ~20s deterministic.
