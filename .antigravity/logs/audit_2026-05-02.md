### Batch 1 тАФ Pipeline persona bug fixes (commit `3b0d093`)

Three bugs prevented pipeline personas from loading any data:

**Bug 1: `bootloader.get_default_project()` wrong project.**
Previously returned the first filesystem glob result. Fixed to check `fixed_manifest` from the persona template first.
File: `app/src/bootloader.py` тАФ `get_default_project()`

**Bug 2: `server.active_cfg()` throws when manifest selector hidden.**
Was calling `input.project_id()` directly, which Shiny raises on when the input does not exist (hidden selector for pipeline-static / pipeline-exploration-simple). Fixed to use `_safe_input(input, "project_id", bootloader.get_default_project())`.
File: `app/src/server.py` тАФ `active_cfg()` reactive calc

**Bug 3: Filters accordion shown for pipeline-static.**
`home_theater.py` was always rendering the Filters accordion. Fixed: gates the accordion on `bootloader.is_enabled("interactivity_enabled")`. When false, the accordion is absent entirely (not hidden inside тАФ the panel itself is not created).
File: `app/handlers/home_theater.py`

### Batch 3 тАФ CSS theme externalisation (ADR-055, commit `788d6ff`)

**Before:** `CSS_THEME` was a ~150-line Python string embedded in `app/src/ui.py`.

**After:**
- `config/ui/theme.css` тАФ new file, authoritative base stylesheet, 15 labelled sections. Fixed latent bug: a CSS comment using `#` (Python syntax) replaced with `/* */`.
- `app/src/ui.py` тАФ reads the CSS file via `bootloader.get_theme_css_path()` at module load; injects via `ui.tags.style()`. `CSS_THEME` string removed.
- `app/src/bootloader.py` тАФ new method `get_theme_css_path()`: reads `theme_css` key from persona config (falls back to `"config/ui/theme.css"`).
- All 6 persona templates (`config/ui/templates/*_template.yaml`) тАФ each now has `theme_css: "config/ui/theme.css"` after the `logic_access` line.

**Effect:** Deployments brand the dashboard by creating a custom CSS file and pointing `theme_css:` at it. No Python changes needed.
