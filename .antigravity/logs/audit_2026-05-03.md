### Sidebar toggle CSS fix

- Root cause: `#nav_sidebar .collapse-toggle` CSS had been applying `position: fixed` to the sidebar toggles, but bslib positions `.collapse-toggle` via CSS Grid + `position: absolute` + CSS variable transitions. When both sidebars are collapsed, the grid transition creates a new fixed-containing-block, causing the fixed-positioned toggle buttons to misplace or become unclickable.
- The previous first-rule selector `#main_layout_outer > .bslib-sidebar-layout > .collapse-toggle` was also wrong — `#main_layout_outer` IS the `bslib-sidebar-layout`; the extra intermediate class never matched, so the left toggle had `position: fixed` but no `top/left` coordinates.
- Fix: removed all `position: fixed`, `top`, `left`, `right` overrides. Only `transform: scale(0.75)` (visual size) and `z-index: 1100` (slightly above bslib's default 1000) remain. Bslib's own positioning logic handles placement correctly.

### Gallery recipe_meta.md standard format

All 13 `recipe_meta.md` files and `recipe_template.md` converted to a new standard:
```markdown
## [Recipe Name]

> 📊 Family · 🔢 Data Pattern · 📈 Difficulty

### Section Heading
…
```
- `# Recipe Metadata: X` → `## X` (h2, not h1; drops the "Recipe Metadata:" prefix noise)
- 3 separate `## Family/Pattern/Difficulty` h2 headings → one `> ` blockquote tag strip
- Content section headings: `## Suitability` etc. → `### Suitability` (h3, clearly sub-headings)

CSS §17 heading scale updated to match:
- `h2` = recipe name: `1.0rem/700` — same visual weight as the "📖 Visual Cookbook: Guidance" accordion header
- `h3` = sections: `0.85rem/700/uppercase` — matches sidebar accordion button density
- `blockquote` = tag strip: `0.78rem`, left amber border (`#c8b86a`), slight inset background

## Why

- Gallery sidebar icons: two-icon glitch looked unpolished, was causing user confusion.
- Select-all alignment: "Select all" text was being truncated on narrow sidebar — right-aligned row is both clearer and avoids overflow.
- Dynamic pivot choices: hardcoded lists in gallery_handlers.py would silently miss new families/patterns added to the index — same bug as build_sidebar_ui had before ADR-057.
- Collapsible panes: users want more space for the guidance text when reading a recipe; minimizing the preview frees vertical space without losing it entirely.
- Sidebar toggle: the `position: fixed` override was a well-intentioned attempt to pin toggles to viewport corners but broke bslib's collapse/expand JS when both sidebars were collapsed simultaneously.
- recipe_meta format: `## Family (Purpose): Ranking` renders identically to `## Suitability` — no visual hierarchy between taxonomy metadata and content. The blockquote strip is compact, visually distinct, and machine-parseable for future tooling.

## False starts / non-obvious choices

- First fix for select-all used `justify-content-end` on the outer div — this is insufficient because `ui.input_checkbox()` renders a `<div class="shiny-input-container">` wrapper that has its own width behaviour. `margin-left: auto` on an inner container is the reliable pattern.
- First sidebar toggle fix changed the CSS selector (`> .bslib-sidebar-layout >` was wrong) but kept `position: fixed`. This improved left toggle positioning but still left the re-open-from-both-collapsed bug. Needed a second pass to understand the bslib grid+transition containing-block issue.
- Considered keeping `position: fixed` with pointer-events workarounds — rejected as too fragile.

## Why (session continuation)

- 13 new recipes: fill gaps in distribution, time-series, ranking, and comparison chart families.
- 6-axis taxonomy: 3-axis taxonomy (family/pattern/difficulty) was too coarse for discovery — users needed to filter by plot type (`geom`), message (`show`), and data volume (`sample_size`) independently.
- TAXONOMY_CHEATSHEET: agents kept using wrong icon assignments; a single-file reference prevents the family/pattern/difficulty icon mismatch from recurring.
- Accordion harmonization: Bootstrap `data-bs-toggle="collapse"` created visually disconnected grey header strip that didn't match Gallery accordion style. bslib accordion is the canonical SPARMVET panel pattern (ADR-064).
- CSS rounding inversions: `border-radius: 0 0 7px 7px` on inner navset-card made only the BOTTOM visually rounded while the top appeared flat — inverted expectation. `overflow: hidden` + `border-radius: 8px` on the accordion-item itself fixes the outer shape.
