# Reference Catalogue Template

When adding a new external reference database to provide interpretation context to the pipeline results, please duplicate this template into a new directory (e.g., `config/references/my_new_context/README.md`) and fill in the details.

## Context Definition
**Dataset Name:** `my_reference.tsv`  
**Purpose:** What does this external dataset do? (e.g., "Maps specific pathogen strains to their broader risk categories.")  
**Source / Origin:** Where did this data come from? (e.g., "WHO Priority Pathogens List 2026")  
**Update Frequency:** How often does this need to be updated? (e.g., "Annually" or "When new AMR mappings are published")  

## Wrangling Instructions
To use this reference in a `_wrangling.yaml` rule (like `lookup_join` or `derive_categories`), developers need to know the exact column names available in your `.tsv` file.

**Available Join Keys (Used for `lookup_right`):**
- `Raw_Name_Column`: The exact string generated by the analysis pipeline that you will match against.
  - *Data Formatting Rule:* The reference dataset must be in a pure **1-to-1 mapping format**. Do not use comma-separated lists in your join keys (e.g., `DrugA, DrugB`). Each term must have its own dedicated row.

**Extractable Context Columns (Used for `extract_column` or added via join):**
- `High_Level_Category`: The aggregated visualization class.
- `Risk_Level`: Optional metadata.
- `Notes`: Additional context.
