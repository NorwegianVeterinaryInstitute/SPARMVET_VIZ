# AMR Phenotypes Context

This directory contains external reference databases used to map raw genome/phenotype identifiers to broader dashboard categories.

## `test_amr_categories_phenotype.tsv`
**Context Definition:**
- **Purpose:** Maps specific Amoxicillin/Ampicillin antibiotic resistance predictions to their broader drug class (e.g., `Beta-lactams`). 
- **Source / Origin:** CGE / ResFinder standard output guidelines.
- **Update Frequency:** As needed when new resistance phenotypes are discovered.

**Wrangling Instructions:**
To use this reference in a `derive_categories` rule, use the following exact column string names from the `.tsv` file:

- **Available Join Keys (`lookup_right`):** `Predicted Phenotype` (The exact string generated by the CGE pipeline).
  - *Data Formatting Rule:* The reference dataset must be in a pure **1-to-1 mapping format**. Do not use comma-separated lists in your join keys. Each phenotype term must have its own dedicated row.
- **Extractable Context Columns (`extract_column`):** `AMR Category (Class)` (The aggregated visualization class).
